Kernel Initialization

The entry point of a WEBMOL application is designed for absolute clarity. This standard index.php demonstrates the molecular connection and basic routing engine.

public/index.php 
<?php

// 1. Calling Webmole Library
require_once "../lib/webmodule.php";

// 2. Initialize the Database Molecule
$wm = new WebMol("", "../lib/dbcon.json", "newdb");

// 3. Define Application Constants
define("SERVER", "http://localhost/WEBMOLv01/");

// 4. Request Capture
$page = isset($_REQUEST['page']) ? $_REQUEST['page'] : "";

// 5. Molecular Routing Engine
switch($page) {
    case "welcome":
        $fle = "../core/welcome.php"; 
        break;
    default: 
        $fle = "../home/Welcome.php";
        break;
}

// 6. Render Engine
$wm->page($fle);
?>
Database Injection

The new WebMol() constructor handles the initial JSON-to-PDO handshake. It connects to your molecular storage in one line.

Molecular Dispatcher

Using the switch($page) logic ensures that the framework only loads the specific molecule needed for that request.

Static Rendering

The $wm->page() method renders the logic into a static, high-speed HTML output for the client side.

The Action Molecule

Action pages in WEBMOL handle the heavy lifting. They define data, map it to template keys, and trigger the final render engine.

core/welcome.php 
<?php

// 1. Define Static & HTML Content
$welcomeText = "<hr/><h1>Welcome to WebMol App Framework</h1>";
$infoText    = "Xtext gets the data from the engine";
$XhtmlContent = "This Text for HTML tag Content";
$XhtmlDesc    = ""; // Null assignment

// 2. Map Arguments to Template Keys
$welcomeArguments = [
    ["txt", $welcomeText, "Xtext", "Default"],
    ["txt", $infoText, "Xinfo", "Default"],
    ["txt", $XhtmlContent, "XhtmlContent", "Default"],
    ["txt", $XhtmlDesc, "XhtmlDesc", "Default Description"]
];

// 3. Point to HTML Template Molecule
$welcomeTemplate = "../core/html/welcome.html";

// 4. Render & Dispatch
$mypage = $wm->html($welcomeArguments, $welcomeTemplate);
echo $mypage;

?>

Molecular Rendering

Dynamic Arguments: Pass $N$ number of variables to your template without complex loop logic.

Null-Safe Fallbacks: If XhtmlDesc is null, WEBMOL automatically injects your defined default text.

The HTML Template

WEBMOL looks for placeholders in your welcome.html like [[Xtext]] or [[Xinfo]] and swaps them instantly.

<div>[[Xtext]]</div>
<p>[[Xinfo]]</p>
